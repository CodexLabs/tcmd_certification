Titanium Modules

Add 'missing' features


Narrow use cases


Appcelerator & Community created


Free, one-time, & subscription-based

  • Add features to Titanium not included in the core distribution
  • Typically narrowly focused on adding a specific feature, rather than a broad API addition
  • Some written by Appcelerator, others by the community. Available in a wide range of prices

Module Distribution

Variety of licensing options


Distributed through Marketplace


'Official' Modules Distributed through Helpdesk


Also, GitHub, Blogs, etc.


Variety of distribution methods are used. Modules can be sold or distributed via marketplace.appcelerator.com. Appcelerator-supported modules are available through support.appcelerator.com. Open-source and unofficial modules are often distributed by their authors via GitHub, blogs, and so forth.

Module Quick-Tour

A quick look at some of the available modules

Urban Airship (iOS and Android)

Push notifications service


Titanium deals with the client side only


Urban Airship provides help for the server side, but much of that work is up to you


PayPal (iOS and Android)

Mobile Payments Library


Requires PayPal merchant account


In development, uses sandbox environment, in production uses live servers


Useful for physical purchases or donations
Can’t be used for ebooks, unlocking app features, etc. because those uses would run afoul of Apple’s StoreKit rules

Bump

Device to device communication


Works over wifi


Uses geolocation and requires close proximity


Requires Bump API key set


Have to register for an API key with Bump

SMS (iOS)

Prepare SMS text messages in app


Cannot send via code


Enhanced functionality over launching the SMS app


TV Out (iOS)

Display app contents on TV screen with additional cable


Test scaling on TV screen, will vary in quality


StoreKit (iOS)

In App Purchase API


Requires set up through iTunes Connect portal


Can only be tested on device


Will need to follow Apple docs for user and item creation


Tied to your iTunes account which is why you have to test on device (that linkage isn’t available in the simulator)

Magtek Credit Card Reader (iOS)

Interface to Magtek hardware


Can process card payments


Requires additional account setup for payments


Can only be tested on device with peripheral

Installing Modules

Two install options: for single project or all projects

    Install for single project:
  1. Download zip file
  2. Place in project's root directory
  3. Build once — to unzip and create necessary folders
  4. Add to tiapp.xml and JavaScript
Installing for all projects doesn't add overhead to your projects, so is probably the best option. We'll cover how to update tiapp.xml and JavaScript on an upcoming slide.

Installing Modules, Continued

    Install for all projects:
  1. Download zip file
  2. Unzip to %TITANIUM_INSTALL_DIR%/modules directory
  3. Add to tiapp.xml and JavaScript
Any future projects you develop on this computer would then be able to use the module.

The Titanium install directory varies by platform and operating system version. For example, on OS X Lion, it would be ~/Library/Application Support/Titanium.
